가계부 2023-06-30 ~ 2023-08-01

프로젝트 개요 개발내용
제목 : 가계부(Model2)
기간 : 2023-06-30 ~ 2023-08-01
인원 : 1명
 - 모델2 방식을 사용하여 가계부 구현
- 기본적인 회원정보, 수입/지출, 카테고리 CRUD
- 월별 소비현황 그래프(3가지) 출력
- 메모에서 해시태그 추출하여 출력
- 필터를 활용한 인코딩, 로그인 유효성검사 기능 적용
- 리스너를 활용한 방문자 수 카운트, MariaDB 연결 기능 적용
담당기능
  1. 로그인/로그아웃
    • 쿠키를 활용한 아이디 저장
      - 쿠키에 저장된 아이디가 있다면 세션에 저장하여 하루동안 아이디 저장 기능을 구현했습니다.
    • 세션 저장, 초기화
      -세션을 이용하여 로그인, 로그아웃 기능을 구현했습니다.
  2. 회원가입
    • ajax를 활용한 아이디 중복 검사
      - 회원가입 시 비동기 방식으로 테이블을 조회하여 아이디 중복검사 기능을 구현했습니다.
  3. 가계부
    • 달력
      - Java 기본 API의 Calendar Class 사용하여 달력을 구현하여 가계부를 구성하였습니다.
    • ajax를 활용한 하위 카테고리 선택
      - 가계부 추가, 수정 시 상위 카테고리를 선택하면 비동기식으로 선택된 카테고리의 하위카테고리 값이 넘어오면서 선택할 수 있게 기능을 구현했습니다.
    • 해시태그 생성
      - 가계부 입력시, 메모에서 #가 있다면 그 단어를 추출하여 해시태그로 생성하였습니다.
  4. 필터와 리스너
    • Filter
      - 모든페이지가 utf-8로 인코딩되게 구현하였습니다.
      - 로그인 유효성검사
    • Listener
      - 방문자 수 카운트(현재접속자, 오늘접속자, 누적접속자) 출력하였습니다.
      - MariaDB 연결
  5. 그래프
    • Ajax와 chart.js를 이용하여 그래프 구현
      - 비동기방식으로 통계값을 가져와서 그래프를 구현하였습니다.
      - 월별 총 수입/지출 그래프
      - 최근 3개월 소비 현황 그래프
      - 월별/카테고리별 소비 현황 그래프

쇼핑몰 2023-05-30 ~ 2023-06-23

프로젝트 개요 개발내용
제목 : 쇼핑몰(Model1)
기간 : 2023-05-30 ~ 2023-06-23
인원 : 3명
 - 모델1 방식을 사용하여 쇼핑몰 구현
- 쇼핑몰 내에서 관리자, 회원, 비회원의 기능별 차등
- 관리자에서도 두 등급으로 나누어 등급별 관리 권한 분리
- 관리자는 상품, 주문상태, 문의, 리뷰관리, 고객리스트조회 가능
- 일반회원은 상품구매, 리뷰작성, 장바구니 기능, 정보변경 가능
- 비회원은 상품, 장바구니 이용만 가능
- 포인트 기능 포함
담당기능
담당기능 : 회원가입, 로그인, 로그아웃, 회원정보관리, 사원정보관리, 리뷰관리, 배송지관리, 문의내역관리, 할인상품관리

  1. 회원가입
    - 회원가입 시 ID 중복검사 기능을 구현했습니다.
    - 회원 탈퇴하여도 DB에선 삭제되지않고 비활성상태로 변경되며 탈퇴했던 아이디로는 가입이 불가합니다.
    - 자바스크립트를 이용한 체크박스를 구현하여 약관동의 후 가입이 가능합니다.
    - 고객과 관리자의 통합된 아이디 리스트를 저장하여 사원과 관리자는 아이디가 중복 불가합니다.
  2. 로그인
    • 관리자와 회원을 구분
      - 로그인 시 회원과 관리자 DB를 비교하여 회원 홈페이지와 관리자 홈페이지로 분리되서 연결되도록 하였습니다.
    • Ajax 활용한 휴면계정 알림띄우기 기능 구현
      - 로그인 시 비동기 방식으로 회원테이블을 조회하여 휴면계정일 경우 알림을 띄우는 기능을 구현했습니다.
  3. 회원정보관리
    • 비밀번호 변경
      - 비밀번호 변경 시 최근 3개의 이전 비밀번호를 비교하여, 저장되어있는 비밀번호로는 변경할 수 없게 하였습니다.
  4. 사원정보관리
    • Level 1 / 2
      - 사원관리(권한등급변경, 사원추가)는 상위관리자인 Level 2 관리자만 가능하도록 기능을 구현했습니다.
  5. 배송지관리
    • 주소 API
      - 주소API를 사용하여 주소를 쉽게 검색하여 배송지를 등록,수정하도록 하였습니다.
  6. 할인상품관리
    • 검색기능
      - 할인상품 목록에서 검색어를 통한 필터기능을 구현하였습니다.
    • 같은 제품 할인 기간 중복 체크
      - 할인제품을 추가, 수정시에 같은 제품에 대한 할인기간이 중복되는지 여부를 DB에서 비교하여 확인 후 중복될 시 추가, 수정을 할 수 없게 하였습니다.
